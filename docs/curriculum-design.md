## Design Principles Applied

1. **Communicative grouping** — modules reflect what learners *do* (narrate, contrast, hypothesise) not raw grammar categories
2. **Visible progression** — B1 → B2 → C1 level chips within every module; learners can see where they stand
3. **Pedagogical sequencing** — within each unit, concepts build on each other (e.g. indefinido → imperfecto → contrast → reported speech)
4. **Manageability** — ≤7 modules, 3–4 units each, max 7 concepts per unit; avoids the KwizIQ "300 flat topics" UX failure
5. **Exercise type laddering** — B1 leans on gap_fill + translation (recognition → production); C1 leans on transformation + free_write (full production)
6. **Variety over repetition** — 9 exercises per concept (3 per type) ensures learners encounter the target structure in different contexts and cannot memorise a single sentence
